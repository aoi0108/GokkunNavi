# **👩‍💻 実装**
このゲームは、特別なサーバーを必要とせず、すべてブラウザ上で動作するように作られています。

### **フロントエンド**
* HTML/CSS/JavaScript
### **主要ライブラリ**
* face-api.js
	* 役割：カメラに映った顔をリアルタイムで認識。
	* 仕組み: 顔のパーツ（目、鼻、口など）の位置を特定し、特に「口が開いた」ことを検知して、ゲームクリアの判定を行う。ブラウザ上で軽快に動作するよう、軽量なAIモデルを利用。
* p5.js
	* 役割：ゲームの描画やインタラクションの基盤として導入。
### **動画生成**
* Sora


# **🎮 4つのシチュエーションの異なるゲーム**
この各ゲームには、まず ストーリーを導入する動画が流れる。 動画の終了直後にカウントダウンが始まることで、ユーザーはゲームの展開に気を取られ、 気づいたら服薬が終わっていた、という仕掛けになっている。

# **ℹ️ 遊び方**
現時点では、PCでアプリを使用することを想定しています。

1. 手元に薬と水を準備する
2. 好きなゲームを選択する
3. ゲームの指示に合わせて服薬し、最後に口を開けてクリア（なぜ口を開けることがクリア条件なのか？：口の中に薬が入っていないことを証明するため）

# **🌞 作品の特徴**

1. 服薬時の嚥下のタイミングをエンターテイメント化する！
2. シチュエーションの異なる4つのゲームを通して服薬ができる！
3. 服薬が成功すると歓声が上がる！失敗しても応援してくれて、カウントダウンからやり直せる！


# **💊 開発背景**
私は幼い頃からずっと薬を飲むことが苦手である。これは、薬を口に含んだ後の「嚥下のタイミング」がわからないからだ。嚥下のタイミングを掴めず、口の中に薬の苦味が充満してしまったり、薬を吐き出したりした経験は数えきれないほどある...

服薬をすることは私にとって非常に苦痛で、服薬に対する心理的ハードルがとても高い。（服薬をしたくないから絶対に体調を崩さないよう健康には常に気をつけているし、体調を崩しても極力薬には頼りたくない...）そこで、服薬時の嚥下のタイミングをエンターテイメント化するシステムがあれば良いと考えた。
